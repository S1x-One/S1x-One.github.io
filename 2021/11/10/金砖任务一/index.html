金砖任务一

发表于2021-11-10|更新于2021-11-10
|阅读量:

搭建区块链网络
搭建单群组FISCO-BCOS联盟链
安装ubuntu依赖：
sudo apt install -y openssl curl
创建目录，并下载安装脚本

创建操作目录

cd ~
mkdir -p fisco
cd fisco

下载脚本

curl -#LO https://github.com/FISCO-BCOS/FISCO-BCOS/releases/download/v2.8.0/build_chain.sh && chmod u+x build_chain.sh
##或者
curl -#LO https://osp-1257653870.cos.ap-guangzhou.myqcloud.com/FISCO-BCOS/FISCO-BCOS/releases/v2.8.0/build_chain.sh && chmod u+x build_chain.sh
搭建单群组4节点联盟链
##安全监听地址请用内网IP或特定的外网IP
bash build_chain.sh -l 127.0.0.1:4 -p 30300,20200,8545
##国密版本执行以下
bash build_chain.sh -l 127.0.0.1:4 -p 30300,20200,8545 -g -G

##若考虑到便于开发与体验,p2p默认监听ip为0.0.0.0
bash build_chain.sh -l 127.0.0.1:4 -p 30300,20200,8545 -i 0.0.0.0
##国密版本执行以下
bash build_chain.sh -l 127.0.0.1:4 -p 30300,20200,8545 -i 0.0.0.0 -g -G
执行成功则会输出 All completed
启动FISCO-BCOS联盟链
bash nodes/127.0.0.1/start_all.sh

搭建WeBASE-Front前置服务
下载安装包
wget https://osp-1257653870.cos.ap-guangzhou.myqcloud.com/WeBASE/releases/download/v1.5.3/webase-front.zip
解压安装包
unzip webase-front.zip
cd webase-front
拷贝SDK证书文件
将节点所在目录 nodes/￥{ip}/sdk 下所有文件（文件在build_chain时生成）拷贝到当前conf目录，供SDK与节点建立连接时使用（SDK会自动判断是否为国密，且是否使用国密SSL）
链的sdk目录包含了ca.crt, sdk.crt, sdk.key和gm文件夹，gm文件夹包含了国密SSL所需的证书
拷贝命令可使用cp -r nodes/${ip}/sdk/* ./conf/
注，只有在建链时手动指定了-G(大写)时节点才会使用国密SSL
服务启动与停止
启动： bash start.sh
停止： bash stop.sh
检查： bash status.sh
当启动成功时，日志 ./log/WeBASE-Front.log 中会出现 Application() - main run success…

检查FISCO-BCOS联盟链及WeBASE-Front前置服务是否启动
登陆linux服务器，检查部署的4节点fisco进程是否有启动，如果未启动，执行命令启动，再次检查，并将执行的命令和完整的命令执行结果截图提交至工程文档2.2章节中。
检查4节点fisco进程是否启动：
ps -ef | grep -v grep | grep fisco-bcos
启动命令：
bash ../nodes/127.0.0.1/start_all.sh
若3030030303,2020020203,8545~8548端口占用：
netstat -anlp | grep tcp
然后:
kill -9 {所占用端口的进程PID}

登陆linux服务器，查看node2节点是否与其他三个节点正常连接，并将执行的命令和完整的命令执行结果截图提交至工程文档2.2章节中。
查看node2节点是否与其他三节点正常连接：
tail -f nodes/127.0.0.1/node2/log/log* | grep connected
正常情况会不停地输出连接信息，从输出可以看出node0与另外3个节点有连接。

登陆linux服务器，检查node2节点共识是否正常，并将执行的命令和完整的命令执行结果截图提交至工程文档2.2章节中。
检查node2节点共识是否正常(检查日志)：
tail -f nodes/127.0.0.1/node2/log/log* | grep +++
正常情况会不停输出++++Generating seal，表示共识正常。

登陆linux服务器，检查WeBASE-Front进程是否有启动，如果未启动，执行命令启动，再次检查，并将执行的命令和完整的命令执行结果截图提交至工程文档2.2章节中。
检查webase-front进程是否启动：
ps -ef | grep webase.front
启停命令：
启动： bash start.sh
停止： bash stop.sh
检查： bash status.sh
查看日志：
cd webase-front
grep -B 3 “main run success” log/WeBASE-Front.log

查看WeBASE-Front服务默认的5002端口是否正常启动，并将执行的命令和完整的命令执行结果截图提交至工程文档2.2章节中。
说明：服务启动过程预计需要1分钟，请等待服务完全启动后，再执行下面步骤。
查看WeBASE-Front服务默认的5002端口是否正常启动：
netstat -anlp | grep 5002

在PC端chrome浏览器上访问WeBASE-Front，访问地址格式为 http://ip:port/WeBASE-Front 请将完整访问地址，和浏览器显示页面截图提交至工程文档2.2章节中。

基于WeBASE-Front平台，创建一个测试用户，用户名称为tester01，将此用户信息截图，包含地址、公钥、用户名称，并将截图提交至工程文档2.2章节中。

详情请关注：
https://fisco-bcos-documentation.readthedocs.io/zh_CN/latest/docs/installation.html
